
### 目录结构：

[SiteProject]
│  chn.php
│  dev.php
│  doc.php
│  index.php
│  mob.php
│  robots.txt
│  
├─@read : readme files
├─code : 类库,流程代码,配置,模版等; 是网站运行的主要代码
│  ├─adpt : api/驱动类库
│  ├─cfgs : 配置
│  ├─core : 核心类库
│  ├─flow : 后台/会员中心流程脚本
│  └─lang : 多语言包
├─html : 静态文件目录
├─root : 各种入口脚本,及前端常用js,css等; 是web服务器所指向的根目录; 
│  ├─a3rd : 第三方 [登录/支付] 接口
│  ├─plus : 接口插件,ajax入口
│  ├─run : 启动路径及常用入口(最初规划在root根目录)
│  └─tools : 工具,二次开发,安装入口等
├─skin : 模板(每套模版一个子目录)及前端常用js,css等; 
│  ├─adm : 后台管理
│  ├─chn : 中文版
│  ├─demodir : 演示目录
│  ├─dev : 开发文档
│  ├─doc : Guides
│  ├─mob : 手机版
│  ├─umc : 会员中心
│  └─_pub : 公用css,js,logo等
├─ures : 上传资源附件目录; 
├─vary : 可变动态文件(缓存,临时文件)
│  ├─ctpl : Tpl-缓存目录
│  └─dtmp : 缓存/临时文件目录
└─vimp : 导入文件(静态图,第三方组件)
    ├─static : 静态图片,图标,附件,字体等文件; 
    ├─vendor : 第三方PHP代码或组件; 
    └─vendui : 第三方前端代码或组件; 
	
* 纠结的目录(怎样放置合理？)：
 - /code/adpt/wechat/ 微信接口类库
 - /code/adpt/weuser/ 微信接口(扩展)类库
 - /root/a3rd/ 第三方支付,登录(含类库,配置,入口文件)


### (附)启动流程：

* 入口文件(示例)：/doc.php
 - 入口配置, 基本常量
 - 加载初始化文件 require(...'/root/run/_init.php');

* 初始化文件：_init.php
 - 初始化 $_cbase
 - 加载系统路径 ('/code/cfgs/boot/_paths.php');
 - 加载常规配置 ('/cfgs/boot/const.cfg.php')
 - 加载后台设置_score '/dset/_score.cfg.php')
 - 包含自动加载类 ('/core/blib/loader.php')
 - 处理outer运行模式,加载别名函数,类自动加载处理等

* 返回入口文件：
 - 执行类似代码：`$vop = new vopShow();`
 - 由vopShow 显示页面
